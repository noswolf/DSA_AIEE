Module 1: OOP Refresher

Before diving into Linked Lists, let's warm up with the Object-Oriented Programming patterns you'll use. Week 2 introduced class, method, and inheritance. Now we add the key patterns for writing clean, professional data structure code.

Encapsulation Getters & Setters super() & Inheritance self keyword

1.1 — The self Keyword

In Python, self is the first parameter of every instance method. It refers to the specific object being created or used. This is how a method knows which object's data to access. Without it, methods would have no way to distinguish between different instances of the same class.

class BankAccount: def __init__(self, owner, balance): self.owner = owner # 'self.owner' stores data ON this specific object self.balance = balance def show_info(self): print(f"Account of {self.owner}, Balance: {self.balance}") # Creating two different objects from the same class acc1 = BankAccount("Alice", 5000) acc2 = BankAccount("Bob", 1200) acc1.show_info() # Output: Account of Alice, Balance: 5000 acc2.show_info() # Output: Account of Bob, Balance: 1200

Key Point: When you call acc1.show_info(), Python automatically passes acc1 as the self argument. You never pass it manually.

1.2 — Encapsulation & Getters / Setters

Encapsulation means hiding the internal state of an object and exposing it through well-defined methods. This protects data from being modified accidentally. In Python, we use a single underscore (_attr) as a convention to signal "private", and we provide getter/setter methods to safely read or write the value.

class Node: def __init__(self, data): self._data = data # _data is "protected" — don't modify directly self._next = None # Getter — allows reading the value safely def get_data(self): return self._data # Setter — allows updating with validation def set_data(self, new_data): if new_data is not None: # Add validation here self._data = new_data else: raise ValueError("Data cannot be None") def get_next(self): return self._next def set_next(self, node): self._next = node # Usage n = Node(42) print(n.get_data()) # 42 n.set_data(99) print(n.get_data()) # 99

1.3 — super() and Inheritance

Inheritance lets a child class reuse code from a parent class. The super() function is used to call the parent class's method from within the child. This is extremely common in data structure implementations where you build specialized structures on top of base ones.

class LinkedList: def __init__(self): self.head = None self.size = 0 def __len__(self): return self.size # DoublyLinkedList inherits all attributes and methods from LinkedList class DoublyLinkedList(LinkedList): def __init__(self): super().__init__() # Call parent's __init__ to set head and size self.tail = None # DLL adds a tail pointer on top of SLL def info(self): print(f"DLL: size={self.size}, head={self.head}, tail={self.tail}") dll = DoublyLinkedList() dll.info() # DLL: size=0, head=None, tail=None print(len(dll)) # 0 — inherited from LinkedList!

Why super()? Without super().__init__(), the parent's constructor never runs, so self.head and self.size would never be initialized. This causes an AttributeError at runtime.

1.4 — Putting It All Together: A Complete Node Class

Here is how a production-quality Node class for a Linked List looks, applying all three OOP concepts above.

class Node: """A single node in a linked list.""" def __init__(self, data): self._data = data # Encapsulated payload self._next = None # Encapsulated pointer # Getter/Setter for data def get_data(self): return self._data def set_data(self, val): self._data = val # Getter/Setter for next pointer def get_next(self): return self._next def set_next(self, node): self._next = node def __repr__(self): return f"Node(data={self._data})" class SinglyLinkedList: """A basic Singly Linked List implementation.""" def __init__(self): self._head = None self._size = 0 def append(self, data): new_node = Node(data) if self._head is None: self._head = new_node else: current = self._head while current.get_next() is not None: current = current.get_next() current.set_next(new_node) self._size += 1 def __len__(self): return self._size
